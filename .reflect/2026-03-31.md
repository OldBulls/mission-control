### Summary
Feishu tool-scopes.js TOOL_SCOPES is mandatory registration point for any new tool
### Details
TOOL_SCOPES map in tool-scopes.js is checked before every tool invocation. If a tool action is not registered, scope-manager treats it as unauthorized regardless of actual app permissions. This caused feishu_drive_permission to fail with misleading "needs drive:drive or docs:doc" — the actual missing piece was scope mapping, not the permissions themselves.
### Suggested Action
Document "New Feishu tool = add TOOL_SCOPES entry first" as a mandatory step in feishu skill or TOOLS.md
